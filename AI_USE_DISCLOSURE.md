# AI Use Disclosure

## Какие ИИ-инструменты использовались

Использовался Codex / AI assistant для помощи в структурировании проекта, подготовке кода приложения, методологии, README и проверки input-файла.

## Для чего использовались

- проверка соответствия input-файла требованиям ТЗ;
- расширение sample input для Air Astana;
- генерация структуры веб-приложения;
- помощь в формулировке методологии;
- подготовка сценарной логики Base case и Stress case;
- подготовка disclosure и README.

## Какие части кода были сгенерированы ИИ

Основная структура Python web server, HTML/CSS/JavaScript интерфейс, функции расчета quality scorecard, ratios и forecast были подготовлены с помощью ИИ.

## Что проверено вручную

- соответствие компании Air Astana;
- наличие 5 лет данных;
- валюта и масштаб input;
- базовая арифметика баланса;
- наличие CFO, CFI, CFF, Capex, dividends и ключевых строк ОПиУ/баланса;
- логика того, что приложение не делает инвестиционную рекомендацию.

## Где ИИ мог ошибиться

- в интерпретации отдельных строк отчетности, если исходная отчетность имеет особые раскрытия;
- в выборе расчетной себестоимости для авиакомпании;
- в сценарных assumptions для прогноза;
- в описании источников, если локальные файлы нужно заменить публичными ссылками;
- в трактовке 2025 года, если данные являются промежуточными или несопоставимыми.

## Что является собственной аналитической позицией

Финальный вывод должен быть проверен студентом: какие факторы реально объясняют динамику Air Astana, какие зоны качества отчетности требуют внимания, и какие ограничения анализа нужно раскрыть на защите.
